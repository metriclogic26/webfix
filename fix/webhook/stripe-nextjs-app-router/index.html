Stripe Webhook Signature Verification Failed on Next.js App Router

Last updated: April 2026

Terminal / Server Log Error
StripeSignatureVerificationError: No signatures found matching the expsignature for payload. Are you passing the raw request body you received from Stripe? Learn more about webhook signing and signing secrets at https://stripe.com/docs/webhooks/signatures

This error means Stripe's HMAC signature does not match the request body your handler received. The cause is almost always the same: the raw body was parsed or transformed before reaching stripe.webhooks.constructEvent(). Next.js App Router parses request bodies automatically — you must read the raw bytes directly.

The Fix — Next.js App Router (app/api/webhook/route.ts)

Use req.text() to get the raw body string. Never use req.json() in a webhook handler — JSON parsing transforms the body and breaks signature verification.

// app/api/webhook/route.ts
import { NextRequest, NextResponse } from 'next/server';
import Stripe from 'stripe';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);
const webhookSecret = processTRIPE_WEBHOOK_SECRET!;

export async function POST(req: NextRequest) {
  const body = await req.text(); // ✅ raw body — do NOT use req.json()
  const sig = req.headers.get('stripe-signature');

  if (!sig) {
    return NextResponse.json({ error: 'Missing stripe-signature header' }, { status: 400 });
  }

  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(body, sig, webhookSecret);
  } catch (err) {
    console.error('Webhook signature verification failed:', err);
    return NextResponse.json({ error: 'Invalid signature' }, { status: 400 });
  }

  // Handle the event
  switch (event.type) {
    case 'payment_intent.succeeded':
      const paymentIntent = event.data.object;
      // handle payment success
      break;
    default:
      console.log('Unhandled event type:', event.type);
  }

  return NextResponse.json({ received: true });
}
⚠ Do not add export const config = { api: { bodyParser: false } } in App Router — that is Pagessyntax and has no effect here.

Pages Router Fix (pages/api/webhook.ts)

In Pages Router, disable the built-in body parser and read the raw body manually:

// pages/api/webhook.ts
import { NextApiRequest, NextApiResponse } from 'next';
import Stripe from 'stripe';
import { buffer } from 'micro';

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY!);
const webhookSecret = process.env.STRIPE_WEBHOOK_SECRET!;

export const config = {
  api: {
    bodyParser: false, // ✅ required for Pages Router
  },
};

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  if (req.method !== 'POST') {
    return res.status(405).end('Method Not Allowed');
  }

  const buf = await buffer(req); // raw body buffer
  const sig = req.headers['stripe-signature'];

  if (!sig) {
    return res.status(400).json({ error: 'Missing stripe-signature header' });
  }

  let event: Stripe.Event;

  try {
    event = stripe.webhooks.constructEvent(buf, sig, webhookSecret);
  } catch (err) {
    console.error('Webhook signature verification failed:', err);
    return res.status(400).json({ error: 'Invalid signature' });
  }

  // Handle the event
  switch (event.type) {
    case 'payment_intent.succeeded':
      // handle pment success
      break;
    default:
      console.log('Unhandled event type:', event.type);
  }

  return res.status(200).json({ received: true });
}

Why signature verification fails

Stripe signs the webhook payload with HMAC-SHA256 using your webhook secret. The signature is computed over the exact raw bytes of the HTTP request body. If anything transforms those bytes before you call constructEvent() — JSON parsing, re-stringifying, middleware processing — the computed signature will not match Stripe's signature and verification fails.

Common causes:

req.json() instead of req.text() — JSON parsing and re-stringifying can change whitespace, key ordering, or encoding. Always use req.text() in App Router.

Global middleware parsing the body — if you have a middleware.ts that reads the request body for other routes, ensure it skips webhook routes or does not consume the body.

— the secret used in constructEvent() must match the signing secret for that specific webhook endpoint in the Stripe dashboard, not the API key. Each endpoint has its own signing secret.

Testing with wrong secret — when using the Stripe CLI for local testing, use the secret the CLI prints (whsec_...), not the endpoint secret from the Stripe dashboard.

Local testing with Stripe CLI

# Install Stripe CLI and forward events to your local server
stripe listen --forward-to localhost:3000/api/webhook

# The CLI prints a signing secret — use this for local testing:
