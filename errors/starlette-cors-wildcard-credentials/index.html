Starlette CORSMiddleware: allow_origins=['*'] Not Allowed with allow_credentials=True

ValueError: allow_origins=['*'] is not allowed when allow_credentials=True. Starlette's CORSMiddleware will not allow combining allow_origins=['*'] with allow_credentials=True.

The fix: Replace allow_origins=['*'] with an explicit list of your allowed origins.

Why this error happens

Browsers reject Access-Control-Allow-Origin: * when the request includes credentials — cookies, Authorization headers, or TLS client certificates. The wildcard and credentials combination is forbidden by the CORS spec.

Starlette enforces this at the middleware level. It raises a ValueError on startup rather than sending an invalid response that browsers would silently reject.

The fix — FastAPI / Starlette

# Wrong — raises ValueError on startup
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,  # ← conflict
    allow_methods=["*"],
    allow_headers=["*"],
)

# Correct — explicit origin list
app.add_middleware(
    CORSMiddleware,
    allow_origins=[
        "https://app.example.com",
        "https://staging.example.com",
    ],
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE", "OPTIONS"],
    allow_headers=["Content-Type", "Authorization"],
)

Dynamic origins from environment

In production, read origins from an environment variable so you don't hardcode them:

import os
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

origins = os.getenv("ALLOWED_ORIGINS", "http://localhost:3000").split(",")

app.add_middleware(
    CORSMiddleware,
    allow_origins=origins,
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE", "OPTIONS"],
    allow_headers=["Content-Type", "Authorization"],
)

Set in your environment:

ALLOWED_ORIGINS=https://app.example.com,https://staging.example.com

If you don't need credentials

If your API doesn't use cookies or Authorization headers, you can keep the wildcard and remove allow_credentials:

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],        # Fine without credentials
    allow_credentials=False,    # Default — can omit entirely
    allow_methods=["*"],
    allow_headers=["*"],
)
